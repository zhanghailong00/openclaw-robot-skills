# 实习项目经历

## 项目名称

基于 AI Agent 的嵌入式机器人自然语言控制系统

## 实习公司

江苏学蠡信息科技有限公司

## 实习时间

2026年5月 - 2026年8月

## 项目背景

公司生产教育实训箱，包含 4-DOF 机械臂、传送带、摄像头、多种传感器等硬件。传统控制方式需要专业软件或代码，学生使用门槛高。本项目通过部署 AI Agent，让学生通过飞书发消息就能控制硬件设备。

## 技术栈

| 类别 | 技术 |
|------|------|
| 硬件平台 | RK3588 ARM64、4-DOF 机械臂、UVC 摄像头、I2C 传感器 |
| AI 框架 | OpenClaw（龙虾）智能体框架、DeepSeek 大语言模型、YOLOv11 目标检测 |
| 通信协议 | UART 串口、GPIO、I2C、WebSocket |
| 开发语言 | Python、Node.js |
| 工具平台 | Linux、conda、飞书开放平台、Git |

## 主要工作

### 一、Skill 插件开发（核心工作）

设计并开发 12 个 Skill 模块，共 39 个 Python 脚本，覆盖实训箱全部硬件功能：

| Skill 模块 | 脚本数 | 功能 |
|------------|--------|------|
| arm-basic | 7 | 机械臂控制（状态查询、归零、移动、夹爪、抓取放置） |
| vision-detect | 2 | YOLO 视觉检测（摄像头检测、目标识别） |
| coord-transform | 2 | 坐标转换（像素→工作台→机械臂坐标） |
| conveyor-belt | 5 | 传送带控制（传感器检测、启停、等待反馈） |
| output-control | 5 | 输出设备（蜂鸣器、继电器、舵机、LED 灯条、调光 LED） |
| obstacle-sensor | 2 | 障碍物检测（红外光电、超声波测距） |
| environment-sensor | 6 | 环境传感器（声音、加速度、光照、温湿度、空气质量、颜色） |
| display-control | 3 | 显示控制（OLED 文字、OLED 清屏、4 位数码管） |
| input-control | 4 | 输入设备（摇杆、电位器、按键、手势传感器） |
| health-sensor | 1 | 健康传感器（心率血氧） |
| self-check | 2 | 系统自检（硬件状态诊断） |
| arm-delivery | 1 | 送餐流程编排 |

**技术亮点：**
- 设计模块化 Skill 架构，每个 Skill 独立可调用，AI 可自由组合完成复杂任务
- 实现视觉引导抓取完整链路：YOLO 检测 → 像素坐标转换 → 逆运动学解算 → 舵机控制
- 编写合并脚本优化响应速度，将多次 Python 启动合并为单次调用，速度提升 50%

### 二、系统集成与调试

- 在 RK3588 ARM64 平台解决 OpenClaw 兼容性问题（Node.js 内存调优、命令卡死修复）

## 面试话术

### 项目一句话介绍

在 RK3588 嵌入式平台上部署 AI Agent，让学生通过飞书发消息就能控制机械臂，AI 自己理解任务、自己调用硬件接口。我负责开发了 12 个 Skill 模块，设计了 30 个测试用例，测试通过率 100%。

### 难点 1：Skill 架构设计

最初想把所有硬件控制写成一个大脚本，后来发现这样 AI 无法灵活组合。改为按功能分 12 类 Skill，每个 Skill 独立可调用，AI 自主编排。比如"抓取汉堡放到盘子"这个任务，AI 会自动组合视觉检测、坐标转换、机械臂控制三个 Skill。

### 难点 2：响应速度优化

飞书发指令到机械臂动作需要 6-8 秒，分析发现瓶颈是 Python 每次启动都重新加载模块和连接串口。写了一个合并脚本 grab_and_place.py，将多次调用合并为一次，速度提升 50%。

### 难点 3：视觉引导抓取

实现完整视觉引导链路：YOLO 检测物体 → 像素坐标转换 → 工作台坐标 → 机械臂坐标 → 逆运动学解算 → 舵机控制。每一步都需要标定和校准。

### 难点 4：硬件调试

夹爪张开角度过大会碰到箱子边缘，通过分析源码发现默认角度是 30°，但我们写成了 45°。修改为 25° 后解决问题。这个教训让我明白写代码要参考原始参数，不能凭感觉。

### 项目亮点

- 完整的 AI + 嵌入式 + 机器人系统集成
- 12 个 Skill 模块，39 个脚本，覆盖全部硬件功能
- 30 个测试用例，9 种测试类型，通过率 100%
- 发现并修复 4 个关键问题
- 从硬件驱动到上层应用的全栈开发能力

| Skill 架构有什么好处？ | 独立可调用、AI 可灵活组合、易扩展、易维护、便于测试 |
| 怎么验证 AI 的编排能力？ | 设计 30 个测试用例，覆盖单 Skill 调用、多 Skill 编排、自然语言理解、异常处理 |
| 遇到过什么困难？ | 夹爪角度问题、smbus2 兼容性、Python 环境不统一，通过分析源码和调试解决 |
