## 一、优先级排序算法

### 1.1 设计目标

在多个物品需要整理时，决定**先抓哪个、后抓哪个**。

### 1.2 核心思路

采用**加权评分**方式，综合考虑两个因素：
1. **物品类型**：不同类型的物品重要性不同
2. **距离远近**：离中心越近越优先（减少移动距离）

### 1.3 计算公式

```
优先级分数 = 类型分数 × 10 + 距离分数
```

### 1.4 详细实现

#### 类型分数配置

```python
TYPE_PRIORITY = {
    'cola': 4,      # 可乐 - 最高优先级（饮料先放，防止倾倒）
    'hanbao': 3,    # 汉堡 - 中等优先级
    'panzi': 2,     # 盘子 - 较低优先级
    'shutiao': 1,   # 薯条 - 最低优先级
}
```

**为什么这样排序？**
- 饮料（可乐）先放：如果后放，可能会被碰倒洒出
- 主食（汉堡）其次：体积较大，需要稳定放置
- 配餐（薯条）最后：体积小，可以最后处理

#### 距离分数计算

```python
# 机械臂坐标中心大约在 (150, 0)
center_x, center_y = 150, 0

# 计算到中心的距离
distance = math.sqrt((arm_x - center_x)**2 + arm_y**2)

# 距离越近，分数越高（取倒数）
distance_score = 1 / (distance + 1)
```

**为什么按距离排序？**
- 减少机械臂移动距离
- 提高整理效率
- 降低能耗

### 1.5 示例计算

假设检测到3个物品：

| 物品 | 类型分数 | 位置 | 距离 | 距离分数 | 最终分数 | 排名 |
|------|---------|------|------|---------|---------|------|
| 可乐 | 4 | (160, 10) | 10.0 | 0.091 | 40.091 | 1 |
| 汉堡 | 3 | (170, -20) | 22.4 | 0.043 | 30.043 | 2 |
| 薯条 | 1 | (155, 5) | 5.4 | 0.156 | 10.156 | 3 |

**排序结果：** 可乐 → 汉堡 → 薯条

## 二、路径规划算法

### 2.1 设计目标

规划机械臂从**起点到终点**的安全运动路径。

### 2.2 核心思路

采用**安全高度 + 悬停 + 安全位过渡**策略：
1. 转向目标方向（atan2计算）
2. 在安全高度水平移动
3. 到达目标上方25mm悬停
4. 下降到目标位置
5. 执行动作后经过安全位过渡

### 2.3 完整运动轨迹（v2.0重构版）

```
抓取流程：
1. 打开夹爪
2. 转向目标方向（atan2计算）
3. 避障绕行（如果有障碍物）
4. 移动到抓取位置上方（悬停25mm）
5. 下降到抓取位置
6. 夹取
7. 抬起（悬停25mm）
8. 安全位过渡

放置流程：
9. 避障绕行（如果有障碍物）
10. 移动到放置位置上方（悬停25mm）
11. 下降到放置位置
12. 松开
13. 抬起（悬停25mm）
14. 回到安全位
```

### 2.4 详细实现

```python
SAFE_HEIGHT = 80  # 安全高度（mm）

def plan_path(start, end, obstacles=None):
    path = []

    # [1] 如果当前高度低于安全高度，先抬升
    if start[2] < SAFE_HEIGHT:
        path.append({
            'action': 'move',
            'position': [start[0], start[1], SAFE_HEIGHT],
            'comment': '抬升到安全高度'
        })

    # [2] 检查是否需要避障
    need_detour = False
    if obstacles:
        need_detour = self._need_detour(start, end, obstacles)

    # [3] 水平移动到目标上方
    if need_detour:
        # 有障碍物，先绕行
        detour_point = self._calculate_detour_point(start, end, obstacles)
        path.append({
            'action': 'move',
            'position': [detour_point[0], detour_point[1], SAFE_HEIGHT],
            'comment': '绕过障碍物'
        })

    path.append({
        'action': 'move',
        'position': [end[0], end[1], SAFE_HEIGHT],
        'comment': '移动到目标上方'
    })

    # [4] 下降到目标位置
    path.append({
        'action': 'move',
        'position': [end[0], end[1], end[2]],
        'comment': '下降到目标位置'
    })

    return path
```

## 三、避障算法

### 3.1 设计目标

当路径上有障碍物时，**自动绕行**避免碰撞。

### 3.2 核心思路

采用**简单绕行策略**：
1. **检测碰撞**：判断路径是否穿过障碍物
2. **计算绕行点**：在障碍物侧面选择安全点
3. **绕行通过**：先移动到绕行点，再继续前进

### 3.3 碰撞检测算法

#### 原理

将障碍物建模为**球体**，检测路径线段是否与球体相交。

#### 数学公式

```
线段参数方程：P = start + t × (end - start),  t ∈ [0, 1]

球心到线段的最短距离：
1. 计算投影参数 t
2. 找到线段上最近点
3. 计算最近点到球心的距离
4. 如果距离 < 球体半径 + 安全距离，则碰撞
```

#### 代码实现

```python
MIN_SAFE_DISTANCE = 50  # 最小安全距离（mm）

def _line_intersects_sphere(self, start, end, sphere):
    """检查线段是否与球体相交"""
    center = sphere.get('center', [0, 0, 0])
    radius = sphere.get('radius', 30)

    # 线段方向向量
    dx = end[0] - start[0]
    dy = end[1] - start[1]
    dz = end[2] - start[2]

    # 球心到起点的向量
    cx = center[0] - start[0]
    cy = center[1] - start[1]
    cz = center[2] - start[2]

    # 计算投影参数 t
    t = (cx * dx + cy * dy + cz * dz) / (dx * dx + dy * dy + dz * dz + 1e-10)
    t = max(0, min(1, t))  # 限制在 [0, 1] 范围内

    # 线段上最近点
    nearest_x = start[0] + t * dx
    nearest_y = start[1] + t * dy
    nearest_z = start[2] + t * dz

    # 计算距离
    distance = math.sqrt(
        (nearest_x - center[0])**2 +
        (nearest_y - center[1])**2 +
        (nearest_z - center[2])**2
    )

    # 判断是否碰撞
    return distance < (radius + MIN_SAFE_DISTANCE)
```

### 3.4 绕行点计算算法

#### 原理

在障碍物的**侧面**选择一个绕行点，让路径从旁边经过。

#### 选择策略

1. 找到路径上**最近的障碍物**
2. 判断障碍物在路径的**左侧还是右侧**
3. 在**相反侧**选择绕行点

#### 代码实现

```python
def _calculate_detour_point(self, start, end, obstacles):
    """计算绕行点"""
    # 找到最近的障碍物
    nearest_obs = None
    min_distance = float('inf')

    for obs in obstacles:
        center = obs.get('center', [0, 0, 0])
        distance = self._point_to_line_distance(center, start, end)
        if distance < min_distance:
            min_distance = distance
            nearest_obs = obs

    if nearest_obs:
        obs_center = nearest_obs.get('center', [0, 0, 0])
        obs_radius = nearest_obs.get('radius', 30)

        # 使用向量叉积判断障碍物在路径的哪一侧
        cross_product = (end[0] - start[0]) * (obs_center[1] - start[1]) - \
                       (end[1] - start[1]) * (obs_center[0] - start[0])

        if cross_product > 0:
            # 障碍物在左侧，从右侧绕
            detour_y = obs_center[1] - obs_radius - MIN_SAFE_DISTANCE
        else:
            # 障碍物在右侧，从左侧绕
            detour_y = obs_center[1] + obs_radius + MIN_SAFE_DISTANCE

        return [obs_center[0], detour_y, self.safe_height]

    # 默认：直接到目标上方
    return [end[0], end[1], self.safe_height]
```

## 八、总结

### 8.1 核心算法

1. **优先级排序**：加权评分（类型×10 + 距离分数）
2. **路径规划**：安全高度 + 悬停 + 安全位过渡
3. **避障算法**：球体碰撞检测 + 向量叉积绕行（半径30mm）

### 8.2 设计原则

- **安全第一**：所有移动都在安全高度进行
- **模块化**：检测、排序、规划、执行分离
- **可配置**：优先级、安全高度、避障参数可调
- **高性能**：直接调用，减少启动开销
- **易使用**：支持飞书自然语言指令

### 8.3 v3.0 改进总结

| 改进项 | 改动 | 效果 |

| 障碍物半径 | 30mm | 适合实际物品大小 |

| 配置统一 | 统一30mm | 简化配置，易维护 |
