## 一、项目概述

### 1.1 功能定义

物品堆叠处理 Skill 实现了**堆叠物品的自动识别、推倒、检测和整理**。

用户只需说"整理堆叠"，系统就会自动：
1. 检测桌面物品状态
2. 判断是否需要推倒堆叠
3. 执行推倒动作
4. 检测推倒后状态
5. 使用家居整理 Skill 排列物品

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| **自动化** | 无需人工干预，全自动完成 |
| **智能化** | 自动识别堆叠状态，智能推倒 |
| **可扩展** | 支持新物品类型、自定义推倒方式 |
| **可复用** | 复用现有 Skill，开发成本低 |

### 1.3 技术指标

| 指标 | 目标值 | 当前值 |
|------|--------|--------|
| 检测准确率 | >90% | ~80% |
| 推倒成功率 | >95% | ~90% |
| 整理成功率 | >95% | ~90% |
| 支持物品数 | 3 | 3（可乐、汉堡、薯条） |

### 3.2 堆叠判断模块（02_check_stack.py）

#### 功能

- 检测桌面物品
- 判断是否需要推倒
- 输出下一步动作

#### 技术实现

**1. 堆叠判断逻辑**

```python
def should_push_over(detections, target_items):
    """判断是否需要推倒"""
    detected_classes = {d['class'] for d in detections}

    # 如果检测到所有目标物品，不需要推倒
    if target_items.issubset(detected_classes):
        return False

    # 如果检测到的物品数量 < 3，需要推倒
    if len(detected_classes) < 3:
        return True

    return False
```

**2. 判断条件**

| 条件 | 结果 | 说明 |
|------|------|------|
| 检测到3个目标物品 | 不推倒 | 物品已散落 |
| 检测到<3个目标物品 | 推倒 | 物品可能堆叠 |
| 检测到0个物品 | 推倒 | 物品可能被遮挡 |

#### 输入输出

| 项目 | 格式 | 示例 |
|------|------|------|
| 输入 | 无 | - |
| 输出 | JSON | `{"is_stacked": true, "need_push": true, "next_action": "push"}` |

### 3.3 位置计算模块（03_calc_position.py）

#### 功能

- 检测桌面物品
- 计算堆叠中心位置
- 转换为机械臂坐标

#### 技术实现

**1. 计算堆叠中心（像素坐标）**

```python
# 计算所有目标物品的平均位置
sum_cx = sum(d['cx'] for d in target_detections)
sum_cy = sum(d['cy'] for d in target_detections)

avg_cx = sum_cx / len(target_detections)
avg_cy = sum_cy / len(target_detections)
```

**2. 转换为机械臂坐标**

```python
# 调用 coord-transform 的 02_convert.py
cmd = [PYTHON_PATH, COORD_SCRIPT, "--px", str(avg_cx), "--py", str(avg_cy), "--json"]

  "workspace": {"x": 18.3, "y": 90.0},
```

### 3.4 推倒动作模块（04_push_over.py）

#### 功能

- 检测堆叠位置
- 执行推倒动作
- 输出推倒结果

#### 技术实现

**1. 推倒方式：上方左移下降右移推倒**

```python
# 推倒参数
above_height = 100  # 物体上方高度（mm）
left_distance = 20  # 向左移动距离（mm）
down_height = 70    # 下降高度（mm）
right_distance = 80 # 向右推倒距离（mm）

# 步骤1：物体上方100mm
above_pos = [stack_pos[0], stack_pos[1], stack_pos[2] + above_height]

# 步骤2：水平向左移动20mm（Y轴负方向）
left_pos = [stack_pos[0], stack_pos[1] - left_distance, stack_pos[2] + above_height]

# 步骤3：下降高度70mm
down_pos = [stack_pos[0], stack_pos[1] - left_distance, stack_pos[2] + above_height - down_height]

# 步骤4：向右移动80mm（推倒，Y轴正方向）
right_pos = [stack_pos[0], stack_pos[1] - left_distance + right_distance, stack_pos[2] + above_height - down_height]
```

**2. 推倒轨迹图示**

```
侧视图：

    100mm ────●─────●───────────────────────●───── 物体上方100mm
              │     │                       │
              │     │                       │
    30mm ─────┼─────●───────────────●───────┼───── 物体上方30mm
              │     │               │       │
              │     │               │       │
    -31mm ────┼─────┼───────────────┼───────●───── 桌面高度
              │     │               │       │
              └─────┴───────────────┴───────┘
              堆叠  步骤2           步骤4   步骤5
              位置  向左20mm        向右80mm 抬起
```

**3. 推倒动作序列**

```python
# 1. 移动到物体上方100mm
arm.move(above_pos, t=0.75, wait=True)

# 2. 水平向左移动20mm
arm.move(left_pos, t=0.5, wait=True)

# 3. 下降高度70mm
arm.move(down_pos, t=0.5, wait=True)

# 4. 向右移动80mm（推倒）
arm.move(right_pos, t=0.5, wait=True)

# 5. 抬起到安全高度
arm.move([right_pos[0], right_pos[1], SAFE_HEIGHT], t=0.75, wait=True)
```

#### 输入输出

| 项目 | 格式 | 示例 |
|------|------|------|
| 输入 | 堆叠位置 | `[158.8, -19.2, -31.2]` |
| 输出 | JSON | `{"success": true, "push_positions": {...}}` |

## 七、问题与解决方案

### 7.1 检测不到物品

**问题：** YOLO 检测不到物品

**可能原因：**
- 光线问题
- 物品不在视野内
- 摄像头被遮挡

**解决方案：**
- 调整光线
- 移动物品到视野中央
- 移动机械臂到安全位置

### 7.2 推倒失败

**问题：** 机械臂推倒失败

**可能原因：**
- 堆叠位置计算错误
- 机械臂无法到达目标位置
- 推倒参数不合适

**解决方案：**
- 检查坐标转换
- 调整推倒参数
- 检查机械臂工作范围

### 7.3 放置位置重复

**问题：** 多个物品放在同一位置

**可能原因：**
- placed_positions.json 文件损坏
- 位置计算逻辑错误

**解决方案：**
- 删除 placed_positions.json 文件
- 检查位置计算逻辑

## 九、总结

### 9.1 核心算法

1. **堆叠判断**：检测物品数量 < 3 则认为堆叠
2. **推倒方式**：上方左移下降右移推倒
3. **位置计算**：物品中心坐标转换
4. **避免重复**：已放置位置保存到文件

### 9.2 设计原则

- **安全第一**：推倒前移动到安全位置
- **模块化**：检测、判断、推倒、整理分离
- **可配置**：推倒参数、目标物品可调
- **可复用**：复用现有 Skill

### 9.3 已知限制

- 只支持可乐、汉堡、薯条
- 推倒方式固定
- 放置区域固定
